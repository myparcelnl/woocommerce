The MyParcel plugin

Introduction

Using this WooCommerce plug-in, you can create five types of MyParcel shipments:

  • parcel within the Netherlands (home address and PostNL collection locations);
  • parcel within the EU and outside of the EU;
  • letterbox parcel;
  • digital stamp;
  • unfranked label.

Third party plug-ins that we support:

  • WooCommerce table rate plug-in;
  • Bolder elements table rate plug-in.

NB: We do not provide support for these plug-ins!

1. Installation

This manual describes the features of the MyParcel plug-in and how to install the MyParcel plug-in in your webshop. We recommend that you always perform the installation on a test environment and perform a back-up of your webshop before you start. There are two ways of performing the installation:

1.1 Through the WordPress plug-in directory

Take the following steps to install the MyParcel plug-in through the WordPress plug-in directory:

  1. In WordPress, go to the menu Plug-ins > New plug-in.
  2. Type 'MyParcel’ into the search bar and click ‘Search’.
  3. Select the WooCommerce MyParcel plug-in and click ‘Install now’. The plug-in will now be downloaded and installed.
  4. Once the download is complete, click ‘Enable plug-in’.

1.2 Manual upload

  1. Download the zip file for the plug-in from the WordPress site.
  2. In WordPress, go to the menu Plug-ins > New plug-in.
  3. At the top of the screen, click ‘Upload plug-in’.
  4. Select the file that you have just downloaded and click ‘Install now’.
  5. Once the installation is complete, click ‘Enable plug-in’.

1.3 Testing

You can add delivery options to the checkout, such as morning and evening delivery and ‘collect from a PostNL location’. You can configure the delivery options you want to offer in the settings for the MyParcel plug-in. Once you have installed the plug-in, we recommend that you go through the order process and place a test order. This allows you to check that the checkout has the correct settings. You can also export shipments once the plug-in has been configured and test label downloads. Testing is free of charge, as long as you don’t offer the labels to PostNL.

NB: you will be charged for digital stamps straight away.

2. Configuration

In the left-hand menu, go to WooCommerce’ > ‘MyParcel’ to configure the MyParcel plug-in.

2.1 General

API settings

For shipments to end up in the correct account, you will need to enter the API key from your MyParcel account. If you are logged into your MyParcel account, you can find it under the sign > ‘Shop settings’ > ‘General’. You may not be able to see the API key yet, because it needs to be ‘generated' first. Click ‘Generate API key’ and copy the API key.

Label display

Specify here whether you want to download the label as a PDF or open it in a new tab.

Label size

Here, you can specify whether you want to print the label on a standard A4 printer or on a label printer (A6).

Ask for start position when printing

This option allows you to continue printing where you left off last time and means you can use all of stickers on the A4 sheet.

Track & Trace link in email

Enable ‘Track & Trace in email’ to send the barcode and Track & Trace link in the shipping confirmation to your customers. The Track & Trace code will then be added to the 'Order completed and/or sent’ email sent from WooCommerce when an order receives the ‘Completed’ status. We recommend disabling the Track & Trace option in your MyParcel account if you are using this option. Otherwise, you customers will receive multiple emails about their dispatched parcel.

NB: Make sure 'Process shipments immediately' is enabled if you use this option in combination with the ‘Track & Trace in email’ option. Otherwise, the Track & Trace code will not be included in the email to your customer since no barcode was generated at the time of sending.

Track & Trace in account

If your customers can create an account in your webshop, you can use this option to ensure that the link for the order barcode is displayed in the account.

Process shipments immediately

If you enable this option, shipments will be processed immediately once they have been sent to MyParcel. They are then immediately pre-registered and the shipment can no longer be amended in your MyParcel back office. If this option is not enabled, the shipment will be entered as a draft. You can still amend the shipment in your MyParcel account after it has been exported.

Order status automation

This option allows you to automatically change the status of the order when a shipment has been successfully exported. You can select the status your shipments will receive once they have been exported.

Save barcode in a note

Placing the barcode in a note for an order allows other plug-ins to read the barcode easily.

Barcode title

Here you can change the text that will appear before the shipment barcode in the order notes. This text will appear before the barcode in the notes. NB: if you use a link with Effect Connect to receive Bol.com orders in your WooCommerce environment, leave the default Tracking code: value in place to automatically link the barcode back to Bol.com.

Diagnostic tools

You can use the diagnostic tools when there are problems exporting the shipments. This creates log files describing the communication between WooCommerce and MyParcel.

2.2 Default export settings

Link parcel type to shipment method

You can link a shipping method to a parcel type so that the correct label is created in your MyParcel account. We explain this further in the example below.

In the example, you can see that the shipping method, ‘fixed rate’, is linked to the parcel type, ‘parcel’. This means that orders with this shipping method are forwarded to MyParcel as a parcel by default.

In the example, you can see that the shipping method, ‘fixed rate’, is linked to the parcel type, ‘letterbox parcel’. This means that orders with this shipping method are forwarded to MyParcel as a letterbox parcel by default.

In the example, you can see that the shipping method, ‘fixed rate’, is linked to the parcel type, ‘digital stamp’. This means that orders with this shipping method are forwarded to MyParcel as a digital stamp by default.

NB: this is a default setting. Of course, this can still be amended manually for each order.

Link customer email address

You can choose to send the email address from the order to MyParcel. This option must be enabled if you are using MyParcel Track & Trace emails. In your MyParcel account, you can enable or disable this email and customise it in your own style.

Link customer telephone number

If you link the customer’s telephone number with the shipment, the courier can use it to deliver the parcel. This significantly increases the probability of foreign shipments being delivered.

Label description

This options allows you to add a description to the shipment. The description will be printed in the top left-hand corner of the label. You can use it to search for the shipment in the MyParcel shipments summary. Use [ORDER_NR] to specify the order number.

Empty parcel weight

Enter the weight of the empty parcel in grams.

Standard HS code

Enter a goods code that can be used by default when exporting shipments to destinations outside of the EU. You can also enter this code per product. Go to ‘Products’ > select product > ‘Product information’ > ‘Shipping methods’ and enter the code.

Customs shipment type

Specify the type of shipments you are sending outside of the EU. As a general rule, these are commercial goods.

Default country of origin

For shipments to destinations outside of the EU, you needed to specify the country from which your products originate. Enter the country code of the relevant country in this field. For the Netherlands, this is NL.

Automatic export

Enable this feature if you want to automatically export your orders to MyParcel immediately after payment.

2.3 Checkout settings

MyParcel address fields

Enable the feature that allows you to split WooCommerce's standard Address line 1 and Address line 2 into three separate fields for Street, House number and Suffix.

Display delivery day

Enable this feature to specify the expected delivery day for your customers in their order confirmation. You can enable displaying delivery days at the checkout under the PostNL tab.

Enable MyParcel delivery options

Enable this feature to display delivery options at your checkout. You can select the delivery options you want to display under the PostNL tab. When you enable this feature, the settings below will become visible.

Display at

Here you can select whether the delivery options are always displayed, even though some products ordered only fit into a letterbox parcel. This allows customers to select delayed delivery, for example, or delivery to a PostNL location, despite the fact that the shipment fits through the letterbox. Once a delivery option has been chosen, the shipment automatically becomes a parcel. A delivery option cannot be selected for a letterbox parcel.

If you would prefer to display these delivery options only for orders sent as parcels, select Shipping methods linked to parcel. Select the preferred shipping methods under ‘Default export settings’ > ‘Parcel type’ > ‘Parcel’.

Delivery options position

If you would like the checkout to be in a different location, you can configure this through ‘Delivery options position’. You can place the MyParcel checkout after invoice information, shipping information or notes. By experimenting with all of the options and viewing them at your checkout, you can discover what works best for your webshop.

Custom styles (CSS)

The checkout automatically adopts the style of your webshop. If you are familiar with writing code and you want to change the style of the checkout, amend this here by changing the CSS.

Titles

Amend titles at the MyParcel checkout so that they match your webshop.

2.4 PostNL

PostNL export settings

Specify which shipping options you want to add to parcel shipments by default when exporting from WooCommerce. With the ‘insured shipment' shipping option, you can set the minimum sale price for which the shipment should be insured.

Enable PostNL delivery

Enable this feature to offer customers the option of home delivery (morning, standard or evening) at your checkout. NB: if you disable this option and enable 'PostNL pick-up', all of your shipments will be sent to a pick-up location by default when the MyParcel checkout is visible for your customers.

Delivery days

This feature allows you to specify the days on which you deliver parcels to a PostNL location. That way, your customers will be offered the correct delivery days.

Closing time

The closing time feature allows you to specify up to which time orders can still be processed. If you take orders to a PostNL location every day around 5pm, it would be good to process the last order around 4pm. If your customers place an order after this time, they will see a later delivery option. Example: Closing time is at 4pm, meaning you can process the orders on the same day and hand them over on time. If an order is placed after 4pm, the checkout will not display next day delivery.

Processing time

Here, you can configure the number of days you need in order to process an order. For example, if you enter '1', the checkout will not display the next working day but the day after that.

If there is no processing time, enter 0 here.

Delivery days window

This option allows you to specify how far in the future your customers will be able to select a delivery day. In this case it is set to 0 days, but this could also be two or five days. The maximum is 14 days. If the delivery days window is set to 0, the days will no longer be displayed, meaning that the consumer will not be able to select a day.

Delivery options

You can set a surcharge for the various delivery options. The surcharge for the delivery option in question will be charged to your customer through the checkout. You can enter a minus amount, which will also be displayed at the checkout. The surcharge therefore applies in addition to the standard rates that you have configured under 'Shipping methods' in your WooCommerce settings. The example below shows how these surcharges are displayed at the checkout.

Enable PostNL pick-up

Enable this feature to offer ‘Collect from PostNL location' at your checkout. If you only want to offer the ‘Collect from PostNL location’ option, select this option without enabling ‘PostNL delivery’. NB: in this case, we recommend that you only display the checkout under 'Shipping methods linked to parcel' so that your customer only sees this option when they click a specific shipping method. If you do not do this, all of your shipments will be sent to a pick-up location.

2.5 Frequently Asked Questions about the configuration

I have enabled the ‘Evening delivery' option, but this is not displayed at the checkout!

You may have entered an address where PostNL does not offer this option, for example at a business park. Another possibility is that you performed the test on Friday with Saturday as the delivery day. Delivery options such as morning and evening delivery are not offered on Saturdays.

All of my shipments have been stored as 'Collect from PostNL location' by default!

You can solve this by turning on Enable PostNL delivery under the PostNL tab or by making the checkout visible only for shipping methods that are linked to the parcel type 'Parcel' in the Default export settings. Please note that the ‘Display at' option under the Checkout settings tab is set to 'For shipping methods linked to parcel'.

3. Using plug-ins

Once the plug-in is installed, you will see MyParcel icons in the table in your order summary. If you do not see the MyParcel icons, you can click 'Screen options' in the top right-hand corner and select 'Actions', 'Barcode' and 'Destination'. If the screen settings have changed, you will see new columns in the order summary.

3.1 Create one shipping label

There are various ways to create shipping labels. First of all, you need to export your shipments. Select a shipment by clicking the checkbox for the relevant order. In the 'Actions' dropdown menu, select 'MyParcel: Export' and click 'Apply'.

In the 'Actions' column, you can also use the MyParcel buttons below.

Export the shipment to MyParcel

Download the label created

Send an email containing a return label from WooCommerce to your customer

3.2 Create multiple shipping labels

If you want to print or export multiple shipments at the same time, you can do so by ticking the checkboxes in the order list. Once you have made a selection of orders to print, click ‘Actions’ and select ‘MyParcel: Export & Print’ or ‘MyParcel: Export’ from the dropdown menu.

‘MyParcel: Export’ will export the selection to MyParcel. ‘MyParcel: Export & Print’ will export all shipments to MyParcel and then download a PDF. If you have exported the shipments separately, you can print them in the WooCommerce environment using ‘MyParcel: Print’. If you try to print when you have not exported the shipments, nothing will be printed.

3.3 Make changes to one shipment

To amend a shipment, click ‘Parcel’ (as in the screenshot below). When you click ‘Parcel’, a small menu will appear that allows you to edit the shipment.

After selecting your preferred options or changing the shipment type, click ‘Save’. You can then export the selection to MyParcel and create the label.

Order information

You can also create shipments from the order information page. Click an order to go to a page containing more information about the order.

3.4 Shipping information

To view information about your shipment, you do not need to go to your MyParcel account. You can view the information in your order summary in WooCommerce. An ‘i’ will appear next to every order forwarded to MyParcel for which a label has been created. Click the ‘i’ to view more information about the status of the shipment. For more detailed information, you can click the barcode in the barcode column.

3.5 Checkout

Below you can see how the checkout will look in a standard WooCommerce environment. The options will become visible once all of the address information has been entered correctly. Some delivery options are not available for all addresses.

Only the available PostNL locations will be displayed. A location may be full and therefore unavailable due to the pressure on PostNL services, in which case it will not be displayed at the checkout.
